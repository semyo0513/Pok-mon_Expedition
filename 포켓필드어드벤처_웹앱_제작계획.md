# 🎮 포켓 필드 어드벤처 (Pocket Field Adventure)
## 학교 현장체험학습용 GAS 웹앱 제작 계획서

> **버전**: v1.0 (2026-07-20)
> **플랫폼**: Google Apps Script (GAS) 웹앱 + Google Sheets (DB)
> **컨셉**: 포켓몬스터 골드(GBC) 레트로 UI/UX 기반 팀 미션 게임
> **대상**: 학교 현장체험학습 참가 학생 및 인솔 교사(관리자)
> **환경**: 모바일 우선(Mobile-First) 반응형 웹앱

---

## 1. 프로젝트 개요

### 1.1 목적
현장체험학습에서 학생들이 팀을 구성하고, 관리자가 등록한 미션을 수행하며,
실시간 랭킹으로 경쟁하는 **게이미피케이션 활동 플랫폼**을 구축한다.

### 1.2 핵심 컨셉 — "포켓볼을 모아라!"
| 기존 기획 요소 | 포켓몬 골드 컨셉 매핑 |
|---|---|
| 팀 | **트레이너 팀** (팀 이름 = 트레이너명/체육관명) |
| 미션 | **포켓볼 획득 이벤트** (야생 미션 출현!) |
| 미션 선택 | **포켓볼 던지기/열기 연출** (룰렛·뽑기·선착순) |
| 포인트 | **경험치(EXP)** 또는 **배지 포인트** |
| 랭킹보드 | **체육관 리더보드** (트레이너 카드 스타일) |
| 관리자 | **오박사(운영자) 모드** |

### 1.3 설계 원칙
- **No-Cost**: Google 계정만으로 서버 비용 0원 운영 (GAS 무료 할당량 내)
- **Mobile-First**: 세로 화면(360~430px) 기준 설계, 터치 타겟 44px 이상
- **Sheets as DB**: 모든 데이터를 Google Sheets에 저장 → 교사가 시트에서 직접 열람/수정 가능
- **저사양 대응**: 픽셀 아트 기반 경량 에셋, 외부 라이브러리 최소화
- **개인정보 최소화**: 이름 외 개인 식별 정보 미수집, 행사 종료 후 데이터 파기 절차 포함

### 1.4 저작권 유의사항 ⚠
포켓몬 공식 스프라이트·로고·명칭은 저작권 보호 대상이므로 **원본 에셋을 직접 사용하지 않는다.**
- GBC 스타일의 **자체 제작 픽셀 아트**(몬스터볼 유사 구체, 오리지널 마스코트) 사용
- 색상 팔레트·프레임 스타일 등 **분위기(레트로 감성)만 오마주**
- 교내 비영리 활용이라도 배포/공개 시 문제가 될 수 있으므로 CSS/SVG로 직접 그린 에셋 권장

## 2. 시스템 아키텍처

### 2.1 전체 구조
```
[학생/교사 모바일 브라우저]
        │  HTTPS
        ▼
[GAS 웹앱 (doGet/doPost)]
  ├─ index.html (SPA: 화면 라우팅)
  ├─ CSS: GBC 테마 (픽셀 폰트, 4색 팔레트)
  ├─ JS: google.script.run 비동기 호출
  └─ 서버 코드(Code.gs)
        │  SpreadsheetApp / LockService / CacheService
        ▼
[Google Sheets (DB)]
  ├─ Members    (명단)
  ├─ Teams      (팀)
  ├─ Missions   (미션)
  ├─ MissionLog (미션 획득/수행 기록)
  ├─ Points     (포인트 원장)
  ├─ Sessions   (로그인 세션/토큰)
  └─ Config     (행사 설정, 입장 코드)
```

### 2.2 기술 스택
| 계층 | 기술 | 비고 |
|---|---|---|
| 프론트엔드 | HTML5 + CSS3 + Vanilla JS | SPA 방식, 해시 라우팅 |
| 폰트 | 'DungGeunMo'(둥근모꼴) 또는 'NeoDunggeunmo' | 한글 픽셀 폰트, 웹폰트 CDN |
| 백엔드 | Google Apps Script | doGet/doPost + google.script.run |
| DB | Google Sheets | 시트 7개, 헤더 고정 |
| 동시성 제어 | LockService | 선착순 미션·포인트 갱신 시 필수 |
| 캐싱 | CacheService | 랭킹보드 폴링 부하 절감 (TTL 10~15초) |
| 실시간성 | 폴링(Polling) 10~15초 간격 | GAS는 WebSocket 미지원 |

### 2.3 GAS 제약사항과 대응
| 제약 | 내용 | 대응 |
|---|---|---|
| 실행 시간 | 1회 호출 최대 6분 | 모든 API를 1초 내 응답하도록 경량화 |
| 동시 실행 | 동시 실행 30개 제한 | 랭킹 폴링 캐싱, 요청 debounce |
| URL 파라미터 | 리다이렉트 후 URL 유지 안 됨 | SPA + 해시 라우팅으로 해결 |
| 쿠키 제한 | iframe 샌드박스로 쿠키 불안정 | localStorage에 세션 토큰 저장 |
| 쓰기 경합 | 동시 쓰기 시 데이터 손상 위험 | LockService.waitLock(10초) 필수 적용 |

## 3. 데이터베이스 설계 (Google Sheets)

### 3.1 Members (명단)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| member_id | TEXT | 고유 ID (M001…) |
| name | TEXT | 이름 |
| class_info | TEXT | 반 정보 (선택) |
| role | ENUM | admin / leader / member |
| team_id | TEXT | 소속 팀 (미배정 시 공백) |
| pw_hash | TEXT | 팀장 비밀번호 해시 (SHA-256 + salt) |
| pw_set | BOOL | 최초 비밀번호 설정 여부 |
| created_at | DATETIME | 등록 일시 |

### 3.2 Teams (팀)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| team_id | TEXT | 고유 ID (T01…) |
| team_name | TEXT | 팀 이름 (팀장이 입력/선택) |
| leader_id | TEXT | 팀장 member_id |
| emblem | TEXT | 팀 엠블럼 코드 (픽셀 아이콘 8종 중 택1) |
| total_points | NUMBER | 누적 포인트 (Points 원장 합계 캐시) |
| created_at | DATETIME | 팀 생성 일시 |
| status | ENUM | active / disbanded |

### 3.3 Missions (미션)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| mission_id | TEXT | 고유 ID (Q001…) |
| title | TEXT | 미션 제목 (예: 드레스코드 미션) |
| description | TEXT | 상세 설명 |
| points | NUMBER | 성공 시 지급 포인트 |
| slot_count | NUMBER | 등록 슬롯 수 (팀 수만큼) |
| slots_json | JSON | 슬롯별 내용 배열 (예: 드레스코드 8종) |
| acquire_type | ENUM | roulette / gacha / first_come |
| ball_type | ENUM | normal / great / ultra / master (희귀도 연출) |
| open_at | DATETIME | 공개 시작 시각 |
| close_at | DATETIME | 마감 시각 |
| status | ENUM | draft / open / closed |

### 3.4 MissionLog (미션 획득/수행 기록)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| log_id | TEXT | 고유 ID |
| mission_id | TEXT | 미션 참조 |
| team_id | TEXT | 획득 팀 |
| slot_index | NUMBER | 획득한 슬롯 번호 |
| slot_content | TEXT | 획득 내용 (예: "레트로 교복 코드") |
| acquired_at | DATETIME | 획득 시각 |
| result | ENUM | pending / success / fail |
| judged_by | TEXT | 판정 관리자 ID |
| judged_at | DATETIME | 판정 시각 |

### 3.5 Points (포인트 원장 — Append Only)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| point_id | TEXT | 고유 ID |
| team_id | TEXT | 대상 팀 |
| delta | NUMBER | 증감량 (+/-) |
| reason | TEXT | 사유 (미션 성공, 보너스, 페널티) |
| ref_log_id | TEXT | MissionLog 참조 (있을 경우) |
| created_by | TEXT | 처리자 |
| created_at | DATETIME | 처리 시각 |

> 포인트는 **수정 불가 원장(append-only)** 으로 관리 → 분쟁 시 이력 추적 가능.
> Teams.total_points는 원장 합계의 캐시로, 불일치 시 재계산 함수로 복구.

### 3.6 Sessions (세션)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| token | TEXT | 랜덤 64자 토큰 |
| member_id | TEXT | 소유자 |
| role | ENUM | admin / leader / member |
| expires_at | DATETIME | 만료 시각 (기본 12시간) |

### 3.7 Config (행사 설정)
| key | value 예시 | 설명 |
|---|---|---|
| event_name | 2026 가을 현장체험학습 | 행사명 |
| leader_code | GOLD-2026 | 팀장 입장 코드 |
| member_code | SILVER-2026 | 팀원 입장 코드 |
| admin_code | OAK-MASTER | 관리자 가입 코드 |
| team_size_min/max | 4 / 6 | 팀 인원 제한 |
| ranking_visible | TRUE | 랭킹 공개 여부 |
| event_phase | building / mission / closed | 행사 단계 제어 |

## 4. 사용자 역할 및 로그인 설계

### 4.1 역할 정의
| 역할 | 입장 방식 | 권한 |
|---|---|---|
| **관리자(오박사)** | 관리자 코드로 가입 → ID/PW 로그인 | 명단 등록, 미션 CRUD, 성공/실패 판정, 랜덤 배정, 설정 |
| **팀장(트레이너)** | 명단에서 본인 선택 + 팀장 코드 → 최초 PW 설정 | 팀 빌딩, 미션 획득, 팀 현황 확인 |
| **팀원(파트너)** | 팀원 코드만 입력 (비밀번호 없음) | 랭킹보드·팀 진행상황 열람 (읽기 전용) |

### 4.2 로그인 플로우
```
[시작 화면 — 타이틀: "POCKET FIELD ADVENTURE"  ▶PRESS START]
   │
   ├─ 관리자 → 가입(관리자 코드 검증) 또는 로그인
   │
   ├─ 팀장 → ① 명단 리스트에서 본인 이름 선택
   │         ② 팀장 코드 입력 (Config.leader_code 검증)
   │         ③ 최초 로그인: 비밀번호 설정(pw_set=FALSE인 경우)
   │            이후 로그인: 비밀번호 입력
   │         ④ 세션 토큰 발급 → localStorage 저장
   │
   └─ 팀원 → 팀원 코드 입력만으로 게스트 세션 발급 (읽기 전용)
```

### 4.3 보안 설계
- 비밀번호: `SHA-256(salt + password)` 해시 저장 (Utilities.computeDigest)
- 세션: 요청마다 토큰 검증 → 만료/위조 시 로그인 화면으로 리턴
- 서버 검증 원칙: 모든 권한 검사는 **서버(Code.gs)에서 수행** (클라이언트 신뢰 금지)
- 코드 노출 대비: 행사 종료 후 관리자 화면에서 코드 일괄 재발급 기능
- 브루트포스 방지: 로그인 실패 5회 시 5분 잠금 (CacheService 활용)

## 5. 기능 상세 설계

### 5.1 팀 빌딩 (트레이너 팀 결성)

**팀장 화면 플로우**
```
[팀 결성소] → 팀 이름 입력 + 엠블럼 선택(픽셀 아이콘 8종)
           → 명단에서 팀원 선택 (미배정자만 노출, 체크박스)
           → 인원 제한 검증 (Config.team_size_min~max)
           → "팀 결성!" → GBC 스타일 결성 연출 (팀 카드 등장)
```

**핵심 규칙**
- 이미 다른 팀에 배정된 학생은 선택 목록에서 자동 제외 (실시간 조회)
- 동시 선택 경합: LockService로 "먼저 저장한 팀"이 선점, 후발 팀에는 갱신된 목록 반환
- 팀 이름 중복 검사 + 부적절 단어 필터(금지어 목록 Config에 등록)
- 팀 수정: 결성 후에는 팀장이 직접 수정 불가, 관리자만 조정 가능 (분쟁 방지)

**관리자 기능**
- 미배정자 목록 실시간 확인
- **랜덤 배정**: ① 잔여 인원을 기존 팀에 균등 분산 ② 새 팀 자동 생성 중 택1
- 배정 시뮬레이션 미리보기 → 확정 버튼으로 반영
- 팀 해체/팀원 이동 수동 조정

### 5.2 팀 미션 활동 (포켓볼 획득 시스템) ★핵심★

#### A. 미션 게시판 (야생 미션의 풀숲)
- 미션 카드가 **풀숲 타일 위의 포켓볼**로 표시되는 그리드 뷰
- 볼 색상으로 상태 표현:
  - 🔴 열린 미션(획득 가능) — 흔들리는 애니메이션
  - ⚪ 마감/품절 — 회색 처리
  - ✨ 우리 팀 획득 완료 — 반짝임 + "GET!" 배지
- 카드 탭 → 미션 상세(설명, 포인트, 마감 시각, 획득 방식 아이콘)

#### B. 미션 획득 방식 3종 (관리자가 미션별로 설정)

**① 룰렛 (Roulette)**
- 슬롯 내용이 룰렛 휠에 배치 → 팀장이 "포켓볼 던지기" 버튼
- 서버에서 잔여 슬롯 중 랜덤 확정 → 클라이언트는 결과에 맞춰 룰렛 감속 연출
- **공정성**: 결과는 반드시 서버에서 결정, 연출은 클라이언트 담당

**② 뽑기 (Gacha) — 포켓볼 오픈 연출 ★**
- 화면에 뒤집힌 포켓볼 N개(잔여 슬롯 수) 배치
- 팀장이 볼 1개 선택 → **포켓볼 오픈 시퀀스**:
  ```
  볼 흔들림(3회, 긴장감) → 찰칵! 열림 → 흰 섬광 →
  슬롯 내용 공개 ("드레스코드: 레트로 교복을 손에 넣었다!")
  → GBC 스타일 팡파레 텍스트 박스
  ```
- ball_type(normal/great/ultra/master)에 따라 볼 디자인·연출 차등

**③ 선착순 (First Come)**
- 공개 시각(open_at)에 버튼 활성화 → 먼저 누른 팀 순서로 슬롯 선택권 부여
- **LockService 필수**: 동시 탭 시 서버 도착 순서로 엄격 판정
- 순번 안에 든 팀만 잔여 슬롯 중 원하는 것 선택 가능

**공통 규칙**
- 팀당 미션별 1회 획득 (중복 방지: MissionLog 검사)
- 획득 즉시 MissionLog에 pending 기록 → 팀 화면 "가방(획득 미션 목록)"에 표시

#### C. 수행 판정 (관리자)
- 관리자 판정 대기열: pending 로그 목록 (팀명, 미션, 획득 내용, 경과 시간)
- [성공] → Points 원장에 +points 기록, 팀 화면에 "미션 클리어! EXP 획득!" 알림
- [실패] → 사유 입력(선택), 재도전 허용 여부 설정 가능
- 보너스/페널티 포인트 수동 지급 기능 (별도 사유 기록)

### 5.3 랭킹보드 (체육관 리더보드)
- **실시간 순위**: 10~15초 폴링 + CacheService 캐싱
- 표시 항목: 순위, 팀 엠블럼, 팀 이름, 총 포인트, 클리어 미션 수, 최근 획득 배지
- 연출:
  - 1~3위 트레이너 카드 확대 표시 (금/은/동 프레임)
  - 순위 변동 시 "▲2" 픽셀 화살표 애니메이션
  - 포인트 증가 시 숫자 카운트업 효과
- 팀원(게스트)도 열람 가능: 우리 팀 하이라이트 표시
- 관리자 옵션: 랭킹 비공개 모드 (마지막 반전용 — 폐회식 공개 연출 "두구두구..." 카운트다운)

### 5.4 관리자 대시보드 (오박사 연구소)
- 행사 단계 제어: 팀빌딩 → 미션 → 종료 (event_phase 스위치)
- 명단 관리: 직접 입력 / 시트 붙여넣기 일괄 등록 / CSV 업로드
- 미션 관리: 등록·수정·복제·공개/마감, 슬롯 일괄 입력 (줄바꿈 구분)
- 판정 대기열 + 포인트 수동 조정
- 통계: 팀별 미션 참여율, 시간대별 획득 추이 (행사 후 리포트용)
- 데이터 파기: 행사 종료 후 개인정보(이름) 일괄 삭제 버튼

## 7. API 설계 (google.script.run 함수 목록)

| 함수 | 권한 | 설명 |
|---|---|---|
| `login(role, payload)` | 공개 | 역할별 로그인 → 토큰 반환 |
| `setInitialPassword(token, pw)` | 팀장 | 최초 비밀번호 설정 |
| `getMemberList(token)` | 팀장/관리자 | 미배정 명단 조회 |
| `createTeam(token, name, emblem, memberIds)` | 팀장 | 팀 결성 (Lock) |
| `getMissions(token)` | 팀장/팀원 | 미션 게시판 데이터 |
| `acquireMission(token, missionId, slotIndex?)` | 팀장 | 미션 획득 (Lock, 서버 랜덤) |
| `getMyBag(token)` | 팀장 | 우리 팀 획득 미션 목록 |
| `getRanking(token)` | 전체 | 랭킹 (Cache 적용) |
| `adminUpsertMembers(token, rows)` | 관리자 | 명단 일괄 등록 |
| `adminUpsertMission(token, mission)` | 관리자 | 미션 등록/수정 |
| `adminJudge(token, logId, result, memo?)` | 관리자 | 성공/실패 판정 + 포인트 지급 |
| `adminRandomAssign(token, mode)` | 관리자 | 미배정자 랜덤 배정 |
| `adminAdjustPoints(token, teamId, delta, reason)` | 관리자 | 포인트 수동 조정 |
| `adminSetPhase(token, phase)` | 관리자 | 행사 단계 전환 |
| `adminPurgeData(token)` | 관리자 | 행사 후 개인정보 파기 |

**공통 응답 형식**: `{ ok: boolean, data?: any, error?: { code, message } }`

|---|---|---|
